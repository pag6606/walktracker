- (decision) AD-1 Hexagonal-lite + dependency rule: dependencias apuntan al dominio; dominio (Session, LapCounter, Chronometer, MetricsCalculator, CalibrationProfile) sin imports de DOM/infra. Puertos=interfaces en el dominio; adaptadores en el borde. Previene dominio acoplado a framework/DOM y UI llamando storage directo.
- (decision) AD-2 [ADOPTED] Vanilla JS single-file (index.html) + sw.js separado. Sin framework/bundler/CDN runtime. Previene toolchain de build y lock-in. Fuente: ADR-01/NFR-5.
- (decision) AD-3 [ADOPTED] localStorage detras de StoragePort; LocalStorageAdapter implementa. Claves wt:config, wt:sessions, wt:activeSession. Migracion a IndexedDB queda tras el puerto. Previene fuga de acoplamiento de storage al dominio.
- (decision) AD-4 [ADOPTED] Session = aggregate root; invariantes (laps>=0, finalizada inmutable, distancia derivada, perimetro congelado al cierre) se enforcean en el dominio. Previene estado inconsistente y reescritura del historial al recalibrar.
- (decision) AD-5 [ADOPTED] Perimetro DERIVADO lapPerimeterM=strideM*stepsPerLap, nunca almacenado como verdad; config guarda medidas crudas. Previene doble fuente de verdad y drift del historial.
- (decision) AD-6 [ADOPTED] Cronometro wall-clock: elapsedS=(now-startedAt)-totalPausesS via ClockPort/Date.now(); setInterval NO es fuente de verdad. Pausa solo explicita. Previene drift por freeze de timers iOS en background.
- (decision) AD-7 Validacion en la frontera (puerto de entrada): inputs crudos (strideM, stepsPerLap) validados (>0, finitos) antes de materializar el aggregate/calibracion. El dominio permanece siempre-valido. Previene aggregates invalidos.
- (decision) AD-8 [ADOPTED] Recuperacion silenciosa: autosave wt:activeSession cada vuelta+10s; al abrir si existe snapshot se reanuda sola recomputando elapsed desde startedAt (el tiempo cerrado cuenta), sin prompt. Previene perdida de datos en purge iOS y UX ambigua.
- (decision) AD-9 Feedback por AudioPort/WebAudioAdapter (beep); AudioContext se reanuda con el gesto de usuario al pulsar Iniciar. navigator.vibrate NO se usa (no existe en iOS Safari). Fuente: MDN AudioContext resume() + autoplay policy. Previene beep fallido en iOS.
- (decision) AD-10 [ADOPTED] Offline-first via SW cache de app shell (cache-first para index.html/manifest/iconos) en sw.js. Fuente: MDN Service Worker API (secure context, registro origin+path). Previene dependencia de red en runtime.
- (decision) AD-11 [ADOPTED] Despliegue GitHub Pages estatico sobre HTTPS; rutas RELATIVAS (./sw.js, start_url ./, scope ./), .nojekyll. Previene SW/instalacion rota bajo subpath de proyecto.
- (constraint) AGENTS.md (invariante de proyecto): hexagonal, regla de dependencia hacia el dominio, validacion en frontera, GitFlow (sin push directo a main), licencias solo Apache-2.0/MIT, contenedores Podman, exploracion con rg/fd.
- (version) JS ES2020+ vanilla (sin transpile); plataforma iOS Safari (PWA); Service Worker API nativa; Web Audio API nativa (Baseline abr-2021); localStorage nativo. Cero dependencias de runtime.
- (assumption) AudioContext se desbloquea al pulsar Iniciar (gesto de usuario) y permanece activo en foreground; si iOS lo suspende al background, se reanuda al volver a primer plano.
- (assumption) Uso de una sola pestana/instancia (no concurrencia multi-tab sobre el mismo localStorage).
